Heel veel mensen bezoeken elke dag tientallen websites op het internet, maar veel mensen weten of kijken niet naar hoe websites in elkaar zitten. Dat het maken van websites bij wijze van spreken onbegrijpelijke magie is. Het bouwen van websites voelt soms aan als magie, maar het zeker weten is (meestal) geen onbegrijpelijke chaos. Dit artikel gaat over hoe websites gecodeert worden en het verschil tussen front-end en back-end.

What is HTML?

De basis van hoe elke website eruit ziet wordt uitgevoerd met de taal HTML, een afkorting voor HyperText Markup Language. HTML, zoals veel andere programmeertalen maakt gebruik van tags om verschillende elementen te genereren die zichtbaar zijn op een webpagina zoals simpele tekst en kopjes, tabellen en lijsten, navigatiemenu's en formulieren. Daarnaast heeft het ook nog een belangrijke taak van het opdelen van de pagina in blokken.

CSS: Een website mooi maken

Een pagina met tekst aan de voorkant kan misschien wel informatief zijn, maar het ziet er eigenlijk helemaal niet uit. HTML is niet gemaakt om dingen mooi te maken, de taal is alleen voor her geraamte: de structuur. Om een website opmaak te geven gebruiken we daarom CSS (Cascading Style Sheets). CSS regelt hoe de website eruit ziet. Een aantal genoemde elementen die men kan toevoegen in HTML zullen waarschijnlijk niet zichtbaar zijn wanneer je de pagina bekijkt. Dat komt namelijk omdat HTML elementen geen eigenschappen geeft. CSS daarentegen kan dingen zoals kleur, afmetingen en positie bepalen.
